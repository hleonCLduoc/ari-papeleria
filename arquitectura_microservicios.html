Arquitectura de Microservicios con Spring Cloud

Flujo de Tráfico y Descubrimiento - Ari Papelería E-commerce

Flujo de una Petición

1. El Cliente Inicia la Petición

El cliente (Frontend o Postman) envía una solicitud HTTP al API Gateway en el puerto 8080.

2. API Gateway Consulta a Eureka

El Gateway pregunta al Eureka Server dónde se encuentra el microservicio que necesita.

3. Eureka Responde

El Eureka Server resuelve el nombre del microservicio y devuelve su dirección (ej: localhost:8083).

4. Derivación al Microservicio

El Gateway reenvia la petición al microservicio correcto. Cada uno ejecuta su lógica de negocio.

5. Persistencia Independiente

Cada microservicio gestiona su propia base de datos MySQL. Esto permite independencia y escalabilidad.

6. Comunicación Interna

Si un microservicio necesita datos de otro, utiliza WebClient o Feign Client para comunicarse mediante REST.

Componentes del Sistema

🚪 API Gateway

Puerto: 8080

Función: Punto de entrada único. Enruta peticiones, valida tokens JWT y centraliza la seguridad.

📖 Eureka Server

Puerto: 8761

Función: Directorio de servicios. Registra y descubre la ubicación de todos los microservicios en tiempo real.

⚙️ Microservicios (6)

Puertos: 8081-8086 (dinámicos)

Función: Ejecutan la lógica de negocio independientemente. Cada uno gestiona su dominio específico.

🗄️ Bases de Datos

Tecnología: MySQL

Función: Cada microservicio tiene su propia BD. Garantiza independencia y evita acoplamiento.

Detalle de Componentes

Componente Puerto Descripción
API Gateway 8080 Puerta de entrada única. Enrutamiento dinámico y seguridad centralizada.
Eureka Server 8761 Registro y descubrimiento de microservicios. Mantiene lista actualizada en tiempo real.
ms-catalogo 8081 Gestión de productos y catálogo del e-commerce.
ms-clientes 8082 Gestión de usuarios y perfiles de clientes.
ms-pedidos 8083 Gestión de órdenes y procesamiento de compras. (CON SWAGGER Y PRUEBAS)
ms-promociones 8084 Gestión de descuentos y campañas promocionales.
ms-resenas 8085 Gestión de calificaciones y opiniones de clientes.
ms-notificaciones 8086 Envío de correos y notificaciones a usuarios.

Ventajas de esta Arquitectura

✓ Escalabilidad

Cada microservicio puede escalar independientemente según su carga de trabajo.

✓ Independencia

Las bases de datos separadas evitan acoplamiento entre servicios.

✓ Resiliencia

Si un servicio falla, los demás continúan funcionando normalmente.

✓ Mantenibilidad

Cambios en un servicio no afectan a los otros. Desarrollo independiente por equipos.

✓ Swagger documentado | ✓ Pruebas unitarias implementadas | ✓ Integración con API Gateway funcional
